# 5.23 커밋 메모

## 목적

5.11 커밋 이후 추가 변경사항만 정리한다.  
5.11 당시 문서와 중복되는 내용은 반복하지 않는다.

## 변경사항

### 1. 실제 실행 진입점 정리

- live runtime 실행 진입점을 `app/main.py`로 통일했다.
- 기존 `tools/runtime/run_ros2_dsvt_tracking_node.py` 경로는 제거했다.
- demo용 `app/main.py`, `demo_frames.py`는 제거했다.

현재 실행:

```bash
/home/gh/anaconda3/envs/pedestrian_gt/bin/python app/main.py \
  --topic /lidar_points \
  --perception dsvt \
  --prediction srlstm
```

### 2. PointPillars perception adapter 추가

- `app/perception/adapters/openpcdet_pointpillar.py` 추가
- OpenPCDet PointPillars checkpoint를 config 기본값으로 연결했다.
- 실행 시 `--perception` 옵션으로 DSVT / PointPillars를 선택한다.

```bash
--perception dsvt
--perception pointpillar
```

PointPillars latency 비교 실행 예:

```bash
/home/gh/anaconda3/envs/pedestrian_gt/bin/python app/main.py \
  --topic /lidar_points \
  --perception pointpillar \
  --prediction srlstm \
  --output-dir artifacts/latency_pointpillar
```

### 3. Latency 측정 개선

`latency.csv`에 frame별 단계 시간을 저장한다.

- `pointcloud_ms`
- `perception_ms`
- `tracking_ms`
- `prediction_ms`
- `warning_ms`
- `marker_ms`
- `total_callback_ms`

마지막 두 줄에는 0번 frame을 제외한 요약을 추가한다.

- `avg_excluding_frame0`
- `stage_percent_excluding_frame0`

기존 DSVT latency 결과에서는 point cloud 변환 비중이 크게 나타났다.

### 4. PointCloud2 parsing 최적화

- 기존 `PointCloud2 -> numpy` 변환은 point를 Python loop로 순회했다.
- `x/y/z/intensity`가 float32이고 row padding이 없는 일반 LiDAR layout에서는 `np.frombuffer()` fast path를 사용하도록 변경했다.
- fast path 조건이 맞지 않는 메시지는 `sensor_msgs_py.point_cloud2.read_points_numpy()`로 fallback한다.
- 기대 효과는 `latency.csv`의 `pointcloud_ms` 감소다.

### 5. Output 저장 기본값 변경

- `--output-dir` 기본값을 `None`으로 변경했다.
- 이제 기본 실행은 artifact를 저장하지 않는다.
- 결과 저장이 필요할 때만 `--output-dir`을 명시한다.

```bash
--output-dir artifacts/latency_check
```

### 6. Adaptive 모델 선택 설계 위치

adaptive 모델 선택은 아직 구현하지 않았다.  
구현한다면 `app/main.py`가 아니라 perception adapter 계층에 두는 것이 좋다.

추천 위치:

```text
app/perception/adapters/adaptive.py
app/perception/model_selector.py
```

예상 구조:

```text
app/main.py
-> RealTimePedestrianTrackingPipeline
-> AdaptivePerceptionModel
   -> model_selector
   -> DSVT or PointPillars
```

### 7. main.py 역할 축소

`app/main.py`에 몰려 있던 보조 기능을 분리했다.

- PointCloud2 변환: `app/io/pointcloud.py`
- artifact 저장: `app/io/artifacts.py`
- RViz2 MarkerArray 생성: `app/visualization/rviz_markers.py`

이제 `app/main.py`는 ROS2 subscribe, pipeline 실행 순서, 옵션 처리 중심으로 유지한다.

### 8. Front/Back loop 실험 런타임 추가

기존 `app/main.py`는 유지하고, 실험용 `app/main_front_back.py`를 추가했다.

- front loop: `/lidar_points` callback에서 PointCloud2 변환, perception, pedestrian filter, tracking 수행
- back loop: 새 tracking snapshot event가 발생하면 prediction, planner snapshot, TTC warning, RViz2 marker publish 수행
- 저장 시 `latency_front_back.csv`에 front/backend 시간을 분리해서 기록한다.

실행 예:

```bash
/home/gh/anaconda3/envs/pedestrian_gt/bin/python app/main_front_back.py \
  --topic /lidar_points \
  --perception pointpillar \
  --prediction srlstm \
  --output-dir artifacts/front_back_experiment
```
