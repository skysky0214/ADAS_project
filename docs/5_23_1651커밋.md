## 목적

OpenPCDet 기반 detection 코드를 ADAS 프로젝트 구조에 맞게 정리하고,
PointPillars 실행/시각화 흐름과 repo 내부 config 관리 구조를 추가한 내용을 정리한다.

## 변경사항

### 1. OpenPCDet adapter 구조 정리

- `app/perception/adapters/openpcdet_base.py` 추가
- OpenPCDet 공통 로직을 base adapter로 분리했다.
- 공통 책임은 아래와 같다.
  - point cloud payload 로드
  - OpenPCDet runtime env 설정
  - cfg/checkpoint 로드
  - single-frame dataset 생성
  - inference 실행
  - `DetectedObject` 변환

- `app/perception/adapters/openpcdet_dsvt.py`
  - 기존 DSVT adapter는 최대한 적게 수정했다.
  - 공통 로직은 base로 이동하고, 현재는 얇은 wrapper 역할만 유지한다.

- `app/perception/adapters/openpcdet_pointpillar.py`
  - PointPillars adapter도 같은 base를 상속하는 구조로 정리했다.

현재 상속 구조:

```text
PerceptionModel
-> OpenPCDetBasePerceptionModel
   -> OpenPCDetDSVTPerceptionModel
   -> OpenPCDetPointPillarPerceptionModel
```

### 2. PointPillars 실행 스크립트 추가

- `tools/perception/run_openpcdet_pointpillar_frame.py` 추가
- PointPillars 단일 프레임 detection 결과를 빠르게 확인할 수 있게 했다.
- 입력은 `.npy` 또는 `.bin` point cloud 파일이다.

실행 예:

```bash
cd /home/sunny/ADAS_project
python3 tools/perception/run_openpcdet_pointpillar_frame.py \
  /home/sunny/OpenPCDet/data/custom/points/000000.npy
```

### 3. Open3D tracking 시각화 스크립트 추가

- `tools/tracking/visualize_openpcdet_tracking_open3d.py` 추가
- 파일 기반 point cloud 입력에 대해 아래 흐름을 한 번에 확인할 수 있게 했다.

```text
point cloud
-> OpenPCDet detection
-> pedestrian filter
-> pedestrian tracking
-> Open3D visualization
```

추가/정리한 기능:

- `--perception dsvt|pointpillar` 선택 가능
- detection 점과 tracking 결과를 한 화면에서 확인 가능
- tracking line 두께 조절 옵션 추가
- detection 점 반지름 조절 옵션 추가

현재 시각화 규칙:

- pedestrian track line / marker: 파란색
- detection color:
  - `Pedestrian`: 파랑
  - `Cyclist`: 초록
  - `Vehicle` / `Car`: 빨강
  - 기타: 노랑

실행 예:

```bash
cd /home/sunny/ADAS_project
python3 tools/tracking/visualize_openpcdet_tracking_open3d.py \
  /home/sunny/OpenPCDet/data/custom/points \
  --perception pointpillar \
  --ext .npy \
  --fps 5 \
  --track-radius 0.18 \
  --track-marker-radius 0.24 \
  --detection-radius 0.30
```

짧은 smoke test:

```bash
python3 tools/tracking/visualize_openpcdet_tracking_open3d.py \
  /home/sunny/OpenPCDet/data/custom/points \
  --perception pointpillar \
  --ext .npy \
  --fps 5 \
  --max-frames 1
```

### 4. OpenPCDet yaml을 repo 내부로 일부 이동

- `configs/openpcdet/pointpillar_custom_ft.yaml` 추가
- 기존 외부 `OpenPCDet` repo의 custom cfg를 ADAS repo 안으로 복사했다.

- `configs/openpcdet/dsvt_custom_ft.yaml` 추가
- 현재는 빈 파일이다.
- DSVT custom 설정 자리를 먼저 확보하는 목적이다.

이 변경으로 PointPillars custom cfg는 ADAS repo 내부에서 직접 관리한다.

### 5. 기본 경로 설정 정리

- `app/core/config.py` 수정
- 기본 OpenPCDet root를 `/home/sunny/OpenPCDet` 기준으로 정리했다.
- PointPillars 기본 cfg는 repo 내부 경로를 보도록 변경했다.

현재 기본 연결:

```text
DEFAULT_OPENPCDET_ROOT -> /home/sunny/OpenPCDet
DEFAULT_POINTPILLAR_CFG_FILE -> configs/openpcdet/pointpillar_custom_ft.yaml
DEFAULT_POINTPILLAR_CHECKPOINT -> /home/sunny/OpenPCDet/output/pointpillar_custom_ft/pointpillar_ft_v1/ckpt/checkpoint_epoch_30.pth
DEFAULT_DSVT_CFG_FILE -> configs/openpcdet/dsvt_custom_ft.yaml
```

주의:

- `DEFAULT_DSVT_CFG_FILE`은 현재 빈 yaml 파일을 가리킨다.
- 따라서 DSVT를 기본값으로 바로 실행하면 실패할 수 있다.
- DSVT를 실제로 사용할 때는 cfg 내용을 채우거나 실행 시 외부 cfg를 직접 넘겨야 한다.
